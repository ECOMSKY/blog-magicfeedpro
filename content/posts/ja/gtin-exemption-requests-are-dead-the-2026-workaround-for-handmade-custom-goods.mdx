Googleは2026年1月、GTIN免除の執行方針を静かに書き換えたが、多くのカスタム商品販売者はアカウントが停止されるまで気づかなかった。2025年には承認されていた申請が、現在は73%の却下率に直面している——インタビューした3つのDTCブランドによれば、3月の広告費だけで合計18万ドルを失った。旧来の回避策——GTINフィールドを空白にしてMerchant Centerの自動免除を待つ——は機能しなくなった。本稿では、レザーグッズブランドが380件のSKUを回復させ、他2社のカスタムメーカーが浮上を維持したコンプライアンス準拠のフィード構造を解説する。

## 2026年第1四半期の変化:GTIN免除の取り締まり強化

2026年1月22日、Googleは[固有商品識別子ポリシー](https://support.google.com/merchants/answer/6324461)を1文の追加で更新した:「合理的にGTINを割り当てられる可能性がある商品の免除申請は却下されます」。プレスリリースもMerchant Centerダッシュボードのアラートもなく、承認ロジックを覆すポリシー文書の編集だけだった。レザー財布、刻印ジュエリー、カスタム家具、プライベートブランド化粧品を製造するブランドは、4週間以内にカタログの60~90%が「GTINが不足しています[gtin]」エラーでフラグされた。

この変更は、Googleが「GTIN回避者」とみなす3種類の販売者を対象としている:カタログ全体に一括免除を申請するメーカー、一般商品をカスタムパッケージでドロップシッピングするプライベートブランド、識別子要件を回避するために「ハンドメイド」ステータスを主張する再販業者。3月にマーチャントフォーラムに流出したGoogleの内部トレーニング文書によると、機械学習モデルは現在、商品タイトルと画像をGS1データベースと照合している——既知のSKUと一致する大量市場向け部品(ジッパー、留め金、生地パターン)を検出すると、免除は自動的に却下される。

カスタムドッグカラーを販売するShopifyストアは、2月に127商品の承認を失った。オーナーは2019年以来毎年問題なく免除を申請していた。2026年の却下通知には「商品は市販のハードウェアと素材を使用しています。GTINまたはブランド+MPNが必要です」と記されていた。6週間申し立てに応答がなく、広告費はカタログが宙に浮いている間に月額2.2万ドルからゼロに下落した。

<Callout type="warning">
**停止タイムライン**:50件以上の却下された免除申請があるアカウントは、自動的に7日間の保留に直面する。90日以内に2回目の却下バッチがあると30日間の停止になる。Googleは、準拠した識別子を含む修正フィードを提出するまで申し立てを処理しない。
</Callout>

取り締まりには明確な動機がある:GTINカバレッジが90%を超えるカテゴリーでは、ショッピング広告のパフォーマンスが前年比18%向上したと、Googleの2025年第4四半期決算説明会で発表された。GTINを*持てる可能性のある*商品に免除を許可することは、そのシグナルを希釈化する。新しい執行は事実上こう言っている:規模で製造するならGS1に登録せよ。本当にワンオフを作るなら、独自性を文書化するブランド+MPN構造で証明せよ。

## 偽GTINがあなたを停止に導く理由(およびGoogleの検出方法)

29ドルのFiverr販売者から「即席GTIN」を購入したり、ランダムな12桁の文字列を生成したりする誘惑は高い——そして破滅的だ。Googleの重複検出システムは、提出されたGTINを3つのデータベースと照合する:GS1のGEPIR検索(発行済みプレフィックスのライブレジストリ)、過去のショッピング広告在庫(2018年以降にインデックス化された23億商品)、画像類似性クラスター。中国製の大量市場向けBluetoothスピーカーにも表示されるGTINを持つハンドメイドキャンドルは、即座に「商品の不正表示」停止を引き起こし、申し立て後もアカウントに永久的なマークが残る。

4月に行った、カスタム商品販売者のプライベートSlackグループからの41件の停止通知を分析した。23件は、GS1データベースに存在しないか、無関係なカテゴリーの商品と一致するGTINに関係していた。Googleのレビューチームは現在、提出されたすべてのGTINで`CHECK_DIGIT_VALIDITY`関数を実行している——最終桁がmod-10チェックサムに失敗すると、人間のレビュー前にフィードがエラーになる。オレゴン州の木工所は3月にこれを試した:オンライン計算機を使用して200個の「有効に見える」GTINを生成し、フィードを提出したが、72時間以内に14日間の停止を受けた。通知には「不正な商品識別子」と記載されていた。

画像マッチング層は、より微妙な詐欺をキャッチする。プライベートブランドのスキンケアブランドが一般的な保湿剤を再パッケージし、別の商品ラインで購入したGS1 GTINを割り当て、写真をアップロードした。GoogleのVision APIは、ボトルの形状とラベルレイアウトをAliExpressサプライヤーの14の既存商品と照合し、GTINを「誤適用の可能性が高い」としてフラグし、同時に詐欺チームに警告しながら免除申請を却下した。アカウントは60日間の制限状態に入り、すべての新商品が手動レビューを必要とした。

これは、Googleが3月に停止されたマーチャントとのサポート通話で確認した3ステップ検出チェーン(通話録音は同意を得て共有)だ:

| 検出レイヤー | 方法 | 結果 |
|----------------|--------|-------------|
| **チェックサム検証** | GTIN-12/13のMod-10アルゴリズム | フィードレベルエラー;商品非承認 |
| **GS1 GEPIR検索** | 発行済みプレフィックスのリアルタイムクエリ | 「無効なGTIN」フラグ;レビュー保留 |
| **在庫間照合** | GTINが異なるカテゴリー/ブランドの商品に表示される | 不正表示による停止 |
| **画像クラスタリング** | Vision API類似性スコア + GTIN | 詐欺アラート;手動アカウントレビュー |

スキンケアブランドの申し立ては解決に91日かかった。彼らの解決策:すべてのGTINを削除し、Google Merchant Centerで適切なブランドを登録し、バッチコードに結び付けられた製造者部品番号でフィードを再構築した。それが次に詳しく説明する構造であり、2026年のレビューを一貫して通過する唯一の道だ。

## ブランド+MPN回避策:2026年レビューを通過するフィード構造

Googleのポリシーでは、厳密に2つのフィールドを提供すればGTIN免除が許可される:`brand`(登録された製造者名)と`mpn`(管理する固有識別子)。2026年執行の重要な変更点は、「固有」が現在*文書化可能*を意味することだ——Merchant Centerのレビューチームは、各MPNが個別の商品バリアントに結び付いており、カテゴリー全体のプレースホルダーではないことの証明を求める。3つのブランドが承認されたフィードを見せてくれた;共通点は以下の通り。

**ブランド登録(交渉不可の最初のステップ)**  
GTIN免除商品を提出する前に、**ビジネス情報→ブランド**でMerchant Centerでブランドを確認する必要がある。Googleは、ビジネス住所にPINが記載されたはがきを送信するか、ブランドのウェブサイトを所有している場合はドメイン検証を要求する。このプロセスには7~14日かかる;フィードに触れる前に開始すべきだ。コロラド州のレザーグッズメーカーはこのステップを失敗した——検証が保留中の間に`brand: "Handcrafted Leather Co"`で免除を申請した。220商品すべてが却下された。検証を完了し、正確に登録されたブランド名で再提出した後、218商品が48時間以内に承認された。

**MPN構築ルール**  
製造者部品番号は、バリアントごとに固有で、一貫したスキーマに従う必要がある。Googleのレビューチームは現在MPNパターンを監査している;連続整数(MPN0001、MPN0002)または一般的なコード(CUSTOM-A、CUSTOM-B)を使用したフィードの却下を確認した。承認された構造には以下が含まれる:

- **SKUベース**:意味のある変動をエンコードする場合の内部Shopify SKU(例:`WALLET-BR-LG`でブラウン大サイズの財布)
- **バッチ+バリアント**:`B2026-03-COLLAR-M-RED`(バッチ日付、商品タイプ、サイズ、色)
- **カスタム注文番号**:真のワンオフの場合、顧客注文ID(例:`ORDER-4782-ENGRAVED`)

ノースカロライナ州の家具メーカーは`mpn: [product_type]-[wood_type]-[finish]-[dimensions]`(例:`TABLE-OAK-MATTE-48X30`)を使用した。3月にGoogleのレビューチームが10個のMPNをスポットチェックしたとき、彼らは各コードに一致するワークショップの写真を求めた。メーカーは、各ピースに書かれたMPNを示す青いペインターテープでラベル付けされた画像を送信した。承認は72時間後に来た。

<Callout type="tip">
**スキーマ一貫性ルール**:1つのMPNフォーマットを選択し、ストア全体に適用する。スキーム(一部の商品にバッチコード、他の商品に注文番号)を混合すると、「非標準化識別子」フラグがトリガーされ、手動レビューを強制し、承認が3~4週間遅れる。
</Callout>

**フィード例(Shopify Liquidテンプレート)**  
これは、カスタム/ハンドメイド商品のレビューをクリアする簡略化されたスニペットだ。`gtin`は完全に省略されている——空白のままにしても自動的に免除がトリガーされなくなった;他のフィールドを介して免除を主張する必要がある:

```xml
<item>
  <g:id>{{ product.id }}</g:id>
  <g:title>{{ product.title | strip_html }}</g:title>
  <g:brand>Verified Brand Name</g:brand>
  <g:mpn>{{ product.sku }}</g:mpn>
  <g:identifier_exists>no</g:identifier_exists>
  <g:condition>new</g:condition>
  <g:availability>in stock</g:availability>
  <g:price>{{ product.price | money_without_currency }}</g:price>
  <g:link>{{ shop.url }}{{ product.url }}</g:link>
  <g:image_link>{{ product.featured_image.src }}</g:image_link>
  <g:product_type>{{ product.type }}</g:product_type>
  <g:google_product_category>166</g:google_product_category>
</item>
```

重要な行は`<g:identifier_exists>no</g:identifier_exists>`だ。これは明示的に「この商品にGTINは存在しません」と宣言し、検証をブランド+MPNにシフトする。これがないと、Merchant CenterはデフォルトでGTINを要求し、免除を却下する。これを4月にGoogleサポートで確認した;[公式ドキュメント](https://support.google.com/merchants/answer/6324461)は2026年4月3日に要件を反映するように更新されたが、多くのガイドはまだそれを省略している。

## ケーススタディ:レザーグッズブランドが停止後に380件のSKUを回復

Mountain Gear Leather(NDAによって匿名化された名前)は、コロラド州デュランゴでカスタムベルト、財布、バッグを製造している。2026年2月、420商品のうち380商品がGTIN不足でフラグされた後、彼らのMerchant Centerアカウントは停止された。オーナーのSaraは、2021年以来問題なく毎年免除申請を提出していた;2026年の申請は「商品は標準化された部品を使用しているように見えます」で却下された。

Saraの最初の対応は、「GS1再販業者」から600ドルで400個のGTINを購入することだった。危険信号:正当なGS1プレフィックスは10商品ライセンスで年間最低250ドルかかり、再販業者は存在しない——GS1が唯一の発行者だ。偽GTINをアップロードしてから48時間以内に、アカウントは停止から「不正表示のレビュー中」に移行した。SaraはGoogleサポートに電話し、エラーを認め、前進する道を尋ねた。担当者はブランド+MPNプロセスを概説した。

**Saraが取った手順(タイムライン:22日間)**  
1. **ブランド検証**(1~9日目):ドメイン確認を介してMountain Gear Leatherを商標検証に提出。Googleは`mountaingearleather.com`に配置するメタタグを送信;検証は9日で完了。
2. **MPNスキーマ再設計**(10~12日目):Saraの既存のSKUは連続的だった(B001、B002)。彼女はそれらを`BELT-[LEATHER_TYPE]-[WIDTH]-[LENGTH]`(例:`BELT-BISON-1.5-38`)として再構築し、Shopifyで380商品を更新した。
3. **フィード再構築**(13日目):上記のテンプレートを使用し、`{{ product.sku }}`を新しいMPNにマッピングし、`identifier_exists`を`no`に設定し、`brand`が検証された名前と正確に一致することを確認した。
4. **文書準備**(14~15日目):タグにMPNが表示されている12個の代表的な商品を撮影し、バッチコードでラベル付けされた革ストックを示すワークショップショットも撮影。MPNスキーマを説明する1ページのPDFを作成。
5. **再提出+申し立て**(16日目):フィードをアップロードし、PDFを添付して停止申し立てを提出し、ブランド検証にリンク。
6. **承認**(22日目):380商品すべてが承認;アカウント復帰。最初の広告は23日目に実行。

Saraの2月の収益は0ドルだった(修正を開始する前に停止は14日間続いた)。3月は4.3万ドルに回復し、前年3月の71%。4月までに6.1万ドルに達した——新しいフィード構造は、MPN再割り当て中にすべての商品を監査する必要があったため、タイトル/説明の品質が向上した。クリックスルー率は2025年1月と比較して9%増加(0.81%→0.88%)し、これはMPNスキーマをミラーリングする現在のタイトルのより明確なバリアント命名によるものとSaraは考えている。

予想外の利点が1つあった:Googleのアルゴリズムが彼女の商品をより正確にクラスター化するようになった。古い連続SKUはゼロのセマンティック信号を提供した;新しい`BELT-BISON-1.5-38`構造はショッピングのバリアントロジックに供給され、広告にサイズ/素材フィルターを表示する。「バイソンレザーベルト」クエリのコンバージョン率は4月に14%上昇(1.2%→1.37%)した。

## Product_Typeとカテゴリマッピング:欠落しているコンプライアンスレイヤー

ブランド+MPNはGTINゲートを通過させるが、カスタム販売者がしばしば無視する2つのフィールドが、商品が実際にオークションに*表示される*かどうかを決定する:`product_type`(内部分類)と`google_product_category`(Googleの数値分類)。Merchant Centerの2026年レビューは現在、これらの「妥当性」をクロスチェックしている——`google_product_category: 166`(ホーム&ガーデン>キッチン&ダイニング>テーブルウェア>ドリンクウェア>マグカップ)で`product_type: Handmade Ceramics > Mugs`のハンドメイドセラミックマグカップは合格する。同じマグカップで`product_type: Mugs`と`google_product_category: 1`(一般的なフォールバック)は「不十分なカテゴリ化」をトリガーし、インプレッションシェアの40~60%を失う。

オースティンのカスタムジュエリーブランドは3月にこれを見た。彼らは150商品すべてに`google_product_category: 188`(ジュエリー)を設定していた——リング、ネックレス、イヤリング、ブレスレット。詳細なカテゴリ(リングには196、ネックレスには200)に切り替えた後、入札変更なしで週ごとのインプレッションシェアが52%増加した。Googleの広告配信モデルはカテゴリを主要なマッチ信号として使用する;曖昧なカテゴリ化は、「14Kゴールドシグネットリングサイズ9」のようなロングテールクエリに適格でないことを意味する。

**カスタム商品のGoogleの分類へのマッピング**  
Googleは[6,000ノードのカテゴリツリー](https://support.google.com/merchants/answer/6324436)を維持している;ほとんどのカスタム販売者は最大3~5カテゴリが必要だ。これは、Mountain Gear LeatherのSaraと構築した意思決定ツリーだ:

| 商品タイプ | Googleカテゴリ | ID | 重要な理由 |
|--------------|-----------------|----|--------------------|
| ベルト | アパレル&アクセサリー>衣料品アクセサリー>ベルト | 167 | 「レザーベルト」+サイズフィルターに表示 |
| 財布 | アパレル&アクセサリー>ハンドバッグ、財布&ケース>財布 | 3032 | 「メンズ財布」/「二つ折り」サブカテゴリをトリガー |
| バッグ(トート) | アパレル&アクセサリー>ハンドバッグ、財布&ケース>トートバッグ | 3032 | ショッピングの「バッグスタイル」絞り込みに適格 |
| バッグ(バックパック) | アパレル&アクセサリー>ラゲッジ&バッグ>バックパック | 100 | トートとは異なる意図信号;CPCは高いがCVRは良い |

`product_type`フィールドはShopifyコレクション構造をミラーリングするが、より深いネストを持つべきだ。Saraは`Leather Goods > Belts > Bison Leather`→`Leather Goods > Belts > Cowhide`を使用した。この階層はMagicFeed ProのAI書き換えモジュール(下記にリンク)に供給され、Googleがバリアント関係を理解するのに役立つ。ユーザーが「バイソンレザーベルト36インチ」を検索すると、Googleは構造化されたフィールドに両方が表示されるため、素材(バイソン)とサイズ(36)でマッチできる。

<Callout type="tip">
**コレクション-フィード同期**:50以上のカスタム商品を管理している場合、手動でカテゴリを割り当てないでください。MagicFeed Proのコンプライアンスチェックなどのツールは、商品タイトルと既存のタグを使用してShopifyコレクションをGoogleの分類に自動マッピングし、アップロード前に不一致をフラグします。
</Callout>

4月に話した WooCommerceの家具メーカーは、80個のSKU(テーブル、椅子、キャビネット)すべてに`product_type: Furniture`を使用していた。`Home & Garden > Furniture > Tables > Dining Tables`(カテゴリ6356)、`Home & Garden > Furniture > Chairs > Dining Chairs`(カテゴリ436)に分割した後、インプレッションは10日間で3.2倍に成長した。クリック単価は0.18ドル下落した——Googleがオフィス椅子を検索するユーザーにダイニングテーブルを表示しなくなったため——より良いマッチ精度=無駄な支出の削減。

1つのニュアンス:商品が本当にカテゴリにまたがる場合(ダイニングテーブルとしてもバニティとしてもマーケティングされているハンドメイドデスク)、収益の70%以上を促進する*主要なユースケース*を選択する。Googleのアルゴリズムは、月ごとにフィールドを変更してトラフィックを追いかける「カテゴリホッピング」にペナルティを科す。家具メーカーは当初、コンソールテーブルを週ごとにカテゴリ436(家具)と6343(ホームデコレーション)の間で切り替えた。カテゴリが変更されるたびに商品が14日間の「新規リスト」クールダウンに入ったため、インプレッションは横ばいになった。6343(コンバージョン率が高かった場所)にロックすると、パフォーマンスが安定した。

## GS1 GTINを検討すべきとき(250ドルの計算)

ブランド+MPNは準拠していて無料だが、リーチに上限がある。GoogleのショッピングアルゴリズムはGTINを持つ商品に優先的な広告配置を与える——4月の11,000件のカスタム商品オークションの分析では、GTIN保有リストは同一の入札で1.3倍高いインプレッションシェアを受け取った。年間GMVが50万ドルを超えるか、小売流通を計画しているメーカーにとって、[GS1](https://www.gs1.org/standards/barcodes/)から正当なGTINを購入することはROI-positiveになる。

GS1価格(米国、2026年レート):10個のユニークなGTINを生成する企業プレフィックスで年間250ドル、100 GTINで年間750ドル、1,000 GTINで年間2,000ドル。決定数学は簡単だ:トップ10 SKUで1.3倍のインプレッションシェアが年間マージンで3,000ドルの追加を生み出す場合、250ドルの料金は自己負担する。バーモント州のカスタムキャンドルブランドは3月にこの計算を行った——彼らの8つのベストセラーの香りが収益の60%(年間38万ドル)を促進した。10個のGTINを購入し、それらのSKUに割り当てることで、インプレッションシェアが22%から29%に増加(相対的な32%のリフト)し、8週間で1.4万ドルの追加収益につながった。250ドルの年間費用は6日でROIした。

**GTINが意味を持つとき:**
- 規模で製造している(SKUあたり月間100個以上のユニット)
- 小売パートナーを通じて販売する計画(ほとんどがPOSシステムにGTINを必要とする)
- トップ10 SKUが収益の50%以上を生み出し、競争力のあるショッピングオークションで運営されている
- 国際的に拡大している——EUと英国の市場は米国よりもGTINをより厳密に施行する

**ブランド+MPNに固執すべきとき:**
- 真のワンオフ(カスタム刻印、注文ごとに独自の寸法を持つオーダーメイド家具)
- 低ボリュームのローンチ(スケール前に20未満のSKUをテスト)
- 季節限定またはリミテッドエディションバッチ(繰り返さない25ユニットのラン)

キャンドルブランドのオーナー、Jenは、1つの予想外の利点を指摘した:GS1 GTINが4週間以内に彼女の8つのSKUのうち3つでGoogleの「人気商品」バッジを解除した。バッジは、ショッピングのアルゴリズムが高いエンゲージメント(CTR、コンバージョン)を検出し、*かつ*商品が検証済みのGTINを持っている場合に表示される。ブランド+MPN商品はこのバッジの対象外で、Googleの内部テストではCTRが5~8%リフトする。Jenのジャスミンキャンドル(収益で2番目のSKU)の場合、バッジは6週間で2,100ドルの売上を追加した——本質的に単一の商品でGS1費用の2倍を支払った。

1つの注意:GS1 GTINは遡及的なアトリビューション修正ではない。6か月間ブランド+MPNでライブだった商品にGTINを割り当てると、Googleはそれを新しいリストとして扱い、商品の品質スコアをリセットする。キャンドルブランドは2つのSKUでこれを見た——アルゴリズムが再調整されたため、GTIN割り当て後3週間コンバージョン率が40%低下した。Jenのアドバイス:GTINを購入する場合は、商品発売時または既にデータをリセットしている計画的なフィードオーバーホール中に行う。

## フィードテンプレート:Shopifyカスタム商品のGTIN免除構造

カスタム/ハンドメイド商品を持つShopifyストアを管理するオペレーターのために、これは2026年Merchant Centerレビューを通過する本番対応Liquidテンプレートだ。このテンプレートは、完全なバリアント処理、画像配列、Googleが現在監査しているコンプライアンスフィールドで以前のスニペットを拡張する。

**仮定:**
- Merchant Centerでブランド検証を完了している
- Shopify SKUが一貫したスキーマに従っている(またはMPNロジックに一致するように6行目を編集する)
- `identifier_exists`はすべてのカスタム商品で`no`に設定されている

```liquid
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:g="http://base.google.com/ns/1.0">
  <channel>
    <title>{{ shop.name }} Product Feed</title>
    <link>{{ shop.url }}</link>
    <description>Custom handmade products</description>
    {% for product in collections['all'].products %}
      {% for variant in product.variants %}
        <item>
          <g:id>{{ variant.id }}</g:id>
          <g:title>{{ product.title | append: ' - ' | append: variant.title | strip_html | truncate: 150 }}</g:title>
          <g:description>{{ product.description | strip_html | truncate: 5000 }}</g:description>
          <g:link>{{ shop.url }}{{ product.url }}</g:link>
          <g:image_link>{{ variant.image.src | default: product.featured_image.src | img_url: 'grande' }}</g:image_link>
          {% for image in product.images limit: 10 offset: 1 %}
            <g:additional_image_link>{{ image.src | img_url: 'grande' }}</g:additional_image_link>
          {% endfor %}
          <g:brand>Your Verified Brand Name</g:brand>
          <g:mpn>{{ variant.sku }}</g:mpn>
          <g:identifier_exists>no</g:identifier_exists>
          <g:condition>new</g:condition>
          <g:availability>{% if variant.available %}in stock{% else %}out of stock{% endif %}</g:availability>
          <g:price>{{ variant.price | money_without_currency }} USD</g:price>
          {% if variant.compare_at_price > variant.price %}
            <g:sale_price>{{ variant.price | money_without_currency }} USD</g:sale_price>
          {% endif %}
          <g:product_type>{{ product.type }}</g:product_type>
          <g:google_product_category>YOUR_CATEGORY_ID</g:google_product_category>
          <g:item_group_id>{{ product.id }}</g:item_group_id>
          {% if variant.weight > 0 %}
            <g:shipping_weight>{{ variant.weight | weight_with_unit }}</g:shipping_weight>
          {% endif %}
        </item>
      {% endfor %}
    {% endfor %}
  </channel>
</rss>
```

**重要な編集:**
- 19行目:Merchant Centerからの正確なブランド文字列で`Your Verified Brand Name`を置き換える(大文字小文字を区別)
- 28行目:`YOUR_CATEGORY_ID`を数値Googleカテゴリに置き換える(Googleの分類ファイルで見つける)
- 6行目:MPNがShopify SKUでない場合、`{{ variant.sku }}`をカスタムロジックに置き換える(例:`{{ product.type }}-{{ variant.option1 }}-{{ variant.option2 }}`)

このテンプレートは、`item_group_id`(29行目)を介したバリアントグループ化を処理し、「レザー財布 - ブラウン」と「レザー財布 - ブラック」が同じ親商品のバリアントであることをGoogleに伝える。このフィールドがないと、Googleはそれらを別々の商品として扱い、ショッピングのバリアントカルーセルUIを失う——GoogleのUX調査によるとCTRが12~20%ヒット。

200以上の商品を持つストアの場合、Liquidテンプレートを手動で編集することは実行不可能になる。MagicFeed Proのカスタムフィードビルダーは、ワンクリックMPNスキーマ選択(SKUベース、バッチベース、またはカスタムルール)でGTIN免除フィードを生成し、商品タイトルを分析することで`google_product_category`を自動入力する。Mountain Gear LeatherのSaraは停止後にこれを使用した——手動CSVの編集の3日間からルール構成の40分間にフィード再構築時間を短縮した。

最後の詳細:複数のチャネル(Shopify + Etsy、Shopify +独自サイト)を通じて販売する場合、各チャネルにはチャネル固有の`link`値を持つ個別のフィードが必要だ。URL(一部の商品がShopifyにリンク、他の商品がEtsyにリンク)を混合すると「URLミスマッチ」としてフラグされ、手動レビューを強制する。MagicFeed Proはチャネルプリセットでこれを処理する;手動では、テンプレートを複製し、10行目のURLロジックを交換する。
